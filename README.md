
# Onboarding Plan Starter

An AI-friendly starter repo for generating a 30-60-90 day onboarding plan from prior notes, agreements, task estimates, and company context.

This repo is designed for operators, consultants, founders, and revenue leaders who want to turn scattered onboarding context into a structured plan without starting from a blank page.

## What this repo is

This is a public starter version of a workflow I use to build onboarding plans from reusable context.

Instead of writing every 30-60-90 from scratch, the idea is to give an AI system:

- prior onboarding documentsrea
- meeting notes and commitments
- your basic operational tasks for intake and vetting a new client
- rough time budgets for recurring tasks
- a clear output structure

From there, the AI can generate a first-pass 30-60-90 that is grounded in prior work, adapted to the current company, and fast to review.

## What's inside

- `AGENTS.md` — instructions for an AI assistant working inside this repo
- `Agreements/30-60-90-template.md` — the output structure for the plan based on past deliverables
- `consulting_pipeline.md` - current next steps for the prospect you're generating a 30-60-90 day plan for
- `CallNotes/sample-notes.md` — synthetic notes from calls and handoff conversations
- `Operations/agenttasks.md` — sample tasks that support vetting and onboarding with dependencies based on prior work
- `Operations/taskdetail.md` — supporting detail for hours estimates based on past engagements

## Who this is for

This starter is useful if you:

- onboard new clients repeatedly
- inherit messy notes across accounts
- need a repeatable way to turn context into an action plan
- want AI to help with synthesis, not hallucination

## How to use it

1. Read `AGENTS.md`.
2. Replace the files in `CallNotes/` with your own sanitized notes, agreements, and company brief.
3. Keep the structure in `Agreements/` and `Operations/` intact.
4. Review the generated draft and edit it like an operator, not a passive approver.

## Recommended workflow

1. Start with recent notes and prior 30-60-90 plans.
2. Add rough time budgets for known task categoriesi in taskdetail.
3. Update your own agenttasks.md to reflect your process
4. Ask the AI to generate a first-pass 30-60-90 using the template.
5. Check whether the plan:
   - matches current business goals
   - reflects realistic sequencing
   - respects time constraints
   - avoids overcommitting in the first 30 days

## Important notes

- Everything in this public repo is synthetic and sanitized.
- Do not upload confidential customer notes, contracts, credentials, or private strategy docs into a public repository.
- Time budgets are planning inputs, not promises.
- AI should propose drafts, not make final delivery commitments for you.

## Suggested adaptations

You can adapt this repo for:

- agency onboarding
- outsourced AE or SDR onboarding
- customer success transitions
- implementation planning
- founder-led sales handoff docs
- internal role ramp plans


## Repo principle

The goal is not to create a perfect plan automatically.

The goal is to make a strong first draft from real context, so your judgment goes into refining the plan instead of reconstructing it from scratch.
